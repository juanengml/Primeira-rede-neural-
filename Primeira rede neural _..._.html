Hora de construir a rede¶

Abaixo voc construir sua rede. Ns construmos a estrutura. Voc implementar o passe para frente e o para trs na rede. Voc tambm definir os hiperparmetros: a taxa de aprendizado, o nmero de unidades ocultas e o nmero de passes de treinamento.

A rede tem duas camadas, uma camada oculta e uma camada de sada. A camada oculta usar a funo sigmide para ativaes. A camada de sada tem apenas um n e usada para a regresso, a sada do n a mesma que a entrada do n. Ou seja, a funo de ativao $ f (x) = x $. Uma funo que recebe o sinal de entrada e gera um sinal de sada, mas leva em considerao o limite, chamada de funo de ativao. Trabalhamos atravs de cada camada de nossa rede, calculando as sadas para cada neurnio. Todas as sadas de uma camada se tornam entradas para os neurnios na prxima camada. Este processo chamado propagao direta .

Usamos os pesos para propagar os sinais da entrada para as camadas de sada em uma rede neural. Usamos os pesos para tambm propagar o erro da sada de volta para a rede para atualizar nossos pesos. Isso chamado de backpropagation .

Hint: You'll need the derivative of the output activation function ($f(x) = x$) for the backpropagation implementation. If you aren't familiar with calculus, this function is equivalent to the equation $y = x$. What is the slope of that equation? That is the derivative of $f(x)$.

Below, you have these tasks:

  1. Implement the sigmoid function to use as the activation function. Set self.activation_function in __init__ to your sigmoid function.
  2. Implement the forward pass in the train method.
  3. Implement the backpropagation algorithm in the train method, including calculating the output error.
  4. Implement the forward pass in the run method.

Treinando a rede¶

Aqui voc configurar os hiperparmetros para a rede. A estratgia aqui encontrar hiperparmetros de forma que o erro no conjunto de treinamento seja baixo, mas voc no est super adaptando aos dados. Se voc treinar a rede por muito tempo ou tiver muitos ns ocultos, ela poder se tornar excessivamente especfica para o conjunto de treinamento e falhar em generalizar para o conjunto de validao. Ou seja, a perda no conjunto de validao comear a aumentar medida que a perda do conjunto de treinamento cair.

Voc tambm usar um mtodo conhecido como Descentramento Estocstico de Gradiente (SGD) para treinar a rede. A ideia que, para cada passe de treinamento, voc pegue uma amostra aleatria dos dados em vez de usar o conjunto de dados inteiro. Voc usa muito mais passes de treinamento do que com descida de gradiente normal, mas cada passe muito mais rpido. Isso acaba treinando a rede de forma mais eficiente. Voc aprender mais sobre o SGD mais tarde.

Escolha o número de iterações¶

Esse o nmero de lotes de amostras dos dados de treinamento que usaremos para treinar a rede. Quanto mais iteraes voc usar, melhor o modelo se ajustar aos dados. No entanto, esse processo pode ter retornos bastante reduzidos e pode desperdiar recursos computacionais se voc usar muitas iteraes. Voc deseja encontrar um nmero aqui em que a rede tenha uma baixa perda de treinamento e a perda de validao seja mnima. O nmero ideal de iteraes seria um nvel que pra logo aps a perda de validao no estar mais diminuindo.

Escolha a taxa de aprendizado¶

Isso dimensiona o tamanho das atualizaes de peso. Se isso for muito grande, os pesos tendem a explodir e a rede no consegue ajustar os dados. Normalmente, uma boa escolha para comear 0,1; no entanto, se voc dividir efetivamente a taxa de aprendizado por n_records, tente comear com uma taxa de aprendizado de 1. Em ambos os casos, se a rede tiver problemas para ajustar os dados, tente reduzir a taxa de aprendizado. Observe que quanto menor a taxa de aprendizado, menores so as etapas nas atualizaes de peso e mais tempo leva para a rede neural convergir.

Escolha o número de nós ocultos¶

Em um modelo em que todos os pesos so otimizados, quanto mais ns ocultos voc tiver, mais precisas sero as previses do modelo. (Um modelo totalmente otimizado poderia ter pesos de zero, afinal.) No entanto, quanto mais ns ocultos voc tiver, mais difcil ser otimizar os pesos do modelo, e mais provvel ser que os pesos abaixo do ideal levem a overfitting. Com overfitting, o modelo memorizar os dados de treinamento em vez de aprender o padro verdadeiro e no generalizar bem os dados no vistos.

Tente alguns nmeros diferentes e veja como isso afeta o desempenho. Voc pode olhar o dicionrio de perdas para uma mtrica do desempenho da rede. Se o nmero de unidades ocultas for muito baixo, o modelo no ter espao suficiente para aprender e, se estiver muito alto, haver muitas opes para a direo que o aprendizado pode tomar. O truque aqui encontrar o equilbrio certo no nmero de unidades ocultas que voc escolher. Geralmente, voc descobrir que o melhor nmero de ns ocultos a usar acaba sendo entre o nmero de ns de entrada e sada.
